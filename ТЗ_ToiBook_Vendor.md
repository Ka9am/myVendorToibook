# ToiBook — Полный план реализации
**Версия:** 2.0  
**Дата:** Апрель 2026  
**Проект:** Веб-маркетплейс вендоров для мероприятий (Казахстан) — как OLX/Авито

---

## 1. Что это

Маркетплейс вендоров для мероприятий в Казахстане (свадьбы, юбилеи, корпоративы). По функционалу — как OLX/Авито, но только для категории «услуги на мероприятия». Два типа пользователей: **вендор** публикует объявления, **клиент** ищет и бронирует.

Всё хранится в `localStorage` — бэкенда нет.

---

## 2. Стек

- **Next.js** (App Router) + **TypeScript**
- **Tailwind CSS**
- Состояние: `localStorage` + React Context для сессии

## 4. Роли и авторизация

Регистрация и вход уже реализованы — не переделывать.

**Сессия:** хранится в `localStorage` как `{ id, name, email, role, companyName? }`, восстанавливается при перезагрузке.

**После входа:**
- Клиент → `/` (каталог)
- Вендор → `/vendor/dashboard`

**Защита роутов:**
- `/vendor/*` — только вендоры, иначе → `/login`
- `/client/*` — только клиенты, иначе → `/login`
- Неавторизованные → `/login`

## 5. Страницы

### 5.1 Публичные

---

#### `/` — Каталог (главная)

- Поиск по названию с живой фильтрацией
- Горизонтальный скролл категорий: Площадки · Тамада · Фото/Видео · Декор · Музыка · Кейтеринг · Другое
- Фильтры: категория, город, цена от/до, рейтинг от
- **Сортировка:** по цене ↑, по цене ↓, по дате (новые), по рейтингу
- Сетка карточек: 3 колонки (desktop) → 2 (tablet) → 1 (mobile)
- Пагинация (по 12 карточек)
- Featured-объявления (поднятые вендором) идут первыми и выделены рамкой

**Карточка оффера:**
- Фото (aspect-ratio 4:3)
- Кнопка ♡ (добавить в избранное) — в правом верхнем углу фото
- Название, город, категория, цена от ₸, рейтинг ★, кол-во отзывов
- Счётчик просмотров 👁
- Бейдж «Featured» если объявление поднято

---

#### `/offers/[id]` — Детальная страница объявления

- Фото-галерея (листать, до 10 фото)
- Название, вендор, город, категория, рейтинг ★ (кол-во отзывов)
- Цена, вместимость
- Описание
- **Характеристики** — блок с иконками и значениями (таблица или сетка), поля зависят от категории
- Кнопка «Поделиться» — копирует ссылку в буфер (copy to clipboard + тост «Ссылка скопирована»)
- Кнопка ♡ «В избранное»
- Счётчик просмотров (увеличивается при каждом открытии страницы)
- Календарь доступных дат
- **Форма заявки** (сайдбар):
  - Выбрать дату
  - Имя (берётся из профиля если авторизован)
  - Кнопка «Отправить заявку» → создаёт booking → переход в чат
  - Если не авторизован → редирект на `/login`
- **Похожие объявления** — блок внизу страницы (та же категория, другие офферы, 4 карточки)
- **Отзывы** — список отзывов с оценкой, именем клиента, датой и текстом

---

#### `/chat/[bookingId]` — Чат

- Шапка: название объявления, имя клиента, дата мероприятия, статус-бейдж
- Пузыри сообщений (свои справа, чужие слева), время под каждым
- Вендор видит кнопки «Подтвердить» (зелёная) / «Отклонить» (серая) если статус `pending`
- При `confirmed` — зелёный баннер «🎉 Вы договорились! Дата: [дата]»
- При `confirmed` и роль `client` — кнопка «Оставить отзыв» (открывает модалку)
- Ввод сообщения + кнопка отправки, Enter = отправить

### 5.3 Вендор (только для вендоров)

---

#### `/vendor/dashboard` — Дашборд

**Счётчики:**
- Всего офферов
- Новых заявок (pending) — с бейджем
- Подтверждённых сделок
- Суммарно просмотров всех офферов

**Последние 5 заявок** с кнопкой «Открыть чат»

**Мои офферы** (список):
- Фото, название, статус (активен / скрыт), просмотры, кол-во заявок

#### `/vendor/create` — Создать объявление

Поля:
- Название оффера (text, обязательное)
- Описание (textarea, обязательное)
- Категория (select: Площадки / Тамада / Фото-Видео / Декор / Музыка / Кейтеринг / Другое)

- Фотографии (drag & drop, до 10 шт, хранить как base64 или Unsplash URL)
- Доступные даты (calendar picker, можно выбрать несколько)
- **Характеристики** — динамический блок, появляется после выбора категории, поля разные для каждой категории (см. раздел 7)

Та же форма, предзаполненная данными оффера.

#### `/vendor/leads` — Все заявки

## 6. Фичи в стиле OLX/Авито

### Избранное
- Кнопка ♡ на каждой карточке и на детальной странице
- Состояние хранится в `localStorage` по `userId`
- Страница `/favorites`

### Счётчик просмотров
- Каждый уникальный визит на `/offers/[id]` инкрементирует счётчик в `localStorage`
- Отображается на карточке (👁 123) и на детальной странице

### Поднять объявление (Featured)
- Вендор нажимает «Поднять» в дашборде
- Оффер получает флаг `featured: true`
- В каталоге: featured-офферы идут первыми, с золотой рамкой и бейджем «⭐ Рекомендуем»

### Статус объявления (активен / скрыт)
- Вендор может скрыть оффер не удаляя (флаг `isActive: false`)
- Скрытые офферы не показываются в каталоге, но остаются в дашборде вендора

### Поделиться
- Кнопка «Поделиться» на детальной странице
- Копирует `window.location.href` в буфер
- Показывает тост «Ссылка скопирована ✓»

### Похожие объявления
- Блок из 4 карточек внизу детальной страницы
- Та же категория, исключая текущий оффер
- Случайный порядок из доступных

### Отзывы и рейтинг
- После `confirmed` booking клиент может оставить отзыв (1 раз на booking)
- Модалка: звёзды 1–5 + текст
- Отзывы отображаются на детальной странице оффера
- Средний рейтинг пересчитывается автоматически и отображается на карточке

### Уведомления (бейдж)
- В Navbar у вендора на «Заявки» — бейдж с количеством `pending` заявок
- Считается из `localStorage` при каждом рендере Navbar

### Сортировка
- Дропдаун в каталоге: «Новые», «Цена ↑», «Цена ↓», «Рейтинг»

## 7. Характеристики объявлений (по категориям)

Как в OLX — у каждой категории свой набор дополнительных полей. Отображаются на детальной странице в виде таблицы «Характеристики», заполняются в форме создания/редактирования оффера.

### Площадки (`venue`)
| Поле | Тип | Значения |
|---|---|---|
| Тип площадки | select | Банкетный зал · Ресторан · Загородная · На природе · Шатёр |
| Площадь | number | кв.м |
| Парковка | boolean | Есть / Нет |
| Своя кухня | boolean | Есть / Нет |
| Кейтеринг | select | Включён · Сторонний разрешён · Только свой |
| Алкоголь | boolean | Разрешён / Нет |

### Тамада (`tamada`)
| Поле | Тип | Значения |
|---|---|---|
| Языки ведения | multiselect | Русский · Казахский · Английский |
| Опыт | number | лет |
| Тип мероприятий | multiselect | Свадьба · Юбилей · Корпоратив · Другое |
| DJ в комплекте | boolean | Да / Нет |
| Звуковое оборудование | boolean | Своё / Нет |

### Фото и видео (`photo_video`)
| Поле | Тип | Значения |
|---|---|---|
| Услуга | select | Только фото · Только видео · Фото + видео |
| Стиль съёмки | select | Репортаж · Постановочный · Смешанный |
| Опыт | number | лет |
| Срок сдачи | number | рабочих дней |
| Дрон | boolean | Есть / Нет |
| Фотокнига | boolean | Включена / Нет |
| Выездная съёмка | boolean | Да / Нет |

### Декор (`decor`)
| Поле | Тип | Значения |
|---|---|---|
| Тип декора | multiselect | Цветочный · Шары · Арки · Столы · Шатры · Световой |
| Монтаж | boolean | Включён / Нет |
| Условие | select | Аренда · Продажа · Аренда или продажа |

### Музыка (`music`)
| Поле | Тип | Значения |
|---|---|---|
| Формат | select | Живая музыка · DJ · Живая + DJ |
| Жанры | multiselect | Казахская · Восточная · Поп · Джаз · Классика · Электронная |
| Кол-во музыкантов | number | чел. |
| Своё оборудование | boolean | Да / Нет |

### Кейтеринг (`catering`)
| Поле | Тип | Значения |
|---|---|---|
| Тип кухни | multiselect | Казахская · Европейская · Восточная · Смешанная |
| Мин. гостей | number | чел. |
| Выездной | boolean | Да / Нет |
| Посуда включена | boolean | Да / Нет |
| Обслуживание | boolean | Официанты включены / Нет |

---

### Реализация характеристик

В модели `Offer` добавить поле `characteristics: Record<string, string | number | boolean | string[]>`.

В форме создания оффера — после выбора категории динамически рендерить нужные поля.

На детальной странице — блок «Характеристики» с иконками и значениями (как таблица или сетка тегов).

## 8. Модели данных

```typescript
type Role = 'client' | 'vendor';
type Category = 'venue' | 'catering' | 'music' | 'photo_video' | 'decor' | 'tamada' | 'other';
type City = 'Almaty' | 'Astana' | 'Shymkent' | 'Aktau' | 'Kostanay';
type BookingStatus = 'pending' | 'confirmed' | 'rejected';

type User = {
  id: string;
  name: string;
  email: string;
  password: string; // хранить хэш или plain (это демо без бэка)
  role: Role;
  companyName?: string;
  city?: string;
  phone?: string;
  createdAt: string;
};

type Offer = {
  id: string;
  vendorId: string;
  vendorName: string;
  title: string;
  description: string;
  category: Category;
  city: City;
  price: number;
  capacity?: number;
  photos: string[];         // base64 или Unsplash URL
  availableDates: string[]; // ISO даты
  characteristics: Record<string, string | number | boolean | string[]>; // доп. поля по категории
  rating: number;           // средний рейтинг, пересчитывается
  reviewCount: number;
  views: number;
  featured: boolean;
  isActive: boolean;
  createdAt: string;
};

type Booking = {
  id: string;
  offerId: string;
  offerTitle: string;
  vendorId: string;
  clientId: string;
  clientName: string;
  eventDate: string;
  status: BookingStatus;
  messages: Message[];
  hasReview: boolean;       // клиент уже оставил отзыв
  createdAt: string;
};

type Message = {
  id: string;
  bookingId: string;
  sender: Role;
  text: string;
  timestamp: string;
};

type Review = {
  id: string;
  offerId: string;
  bookingId: string;
  clientId: string;
  clientName: string;
  rating: number;   // 1–5
  text: string;
  createdAt: string;
};
```

## 8. Моковые данные (seed)

При первом запуске (`localStorage` пуст) засеять:

**Пользователи:**
- 1 клиент: `{ name: 'Айгерим', email: 'client@test.com', password: '123456', role: 'client' }`
- 2 вендора с разными компаниями

**Офферы:** 12–15 штук по всем категориям и городам, с реальными Unsplash URL для фото (тема: wedding, event, flowers, catering). Несколько с `featured: true`, разные цены и рейтинги.

**Bookings:** 2–3 заявки с перепиской (одна `pending`, одна `confirmed`, одна `rejected`).

**Reviews:** 2–3 отзыва к офферам.

## 10. Что должно работать в итоге

1. Каталог с поиском, фильтрами, сортировкой, featured-офферами
2. Детальная страница с галереей, заявкой, похожими офферами, отзывами
3. Чат с подтверждением/отклонением и формой отзыва
4. Вендор: создаёт, редактирует, удаляет, скрывает и поднимает объявления
5. Клиент: видит свои заявки, избранное
6. Вендор: видит leads inbox с бейджем новых заявок
7. Счётчики просмотров на всех офферах
8. Избранное (♡) работает для авторизованного клиента
9. Поделиться ссылкой (copy to clipboard)
10. Профиль с редактированием для обеих ролей
11. Уведомление-бейдж в навбаре
12. Полностью мобильная версия (mobile-first)
13. Всё без бэка — только localStorage
