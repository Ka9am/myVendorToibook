# ТЗ — ToiBook Vendor Platform
**Версия:** 1.0  
**Дата:** Апрель 2026  
**Проект:** Веб-маркетплейс вендоров для мероприятий (Казахстан)

---

## 1. Обзор проекта

**ToiBook** — это веб-маркетплейс в стиле Bridebook, адаптированный для казахстанского рынка. Платформа соединяет клиентов, которые планируют мероприятия (свадьбы, юбилеи, корпоративы), с вендорами, которые предоставляют услуги: площадки, тамады, фотографы, декор, музыка, кейтеринг.

**Аналог:** Bridebook (bridebook.com) — но для Казахстана, с поддержкой казахского и русского языков, и местными особенностями (тамада, казахская кухня, местные города).

## 5. Авторизация

### Страница `/register`
- Поля: Имя, Email, Пароль
- Выбор роли: **Я ищу услуги (Клиент)** / **Я предоставляю услуги (Вендор)**
- Если Вендор: доп. поле «Название компании»
- Кнопка: «Зарегистрироваться»
- Ссылка: «Уже есть аккаунт? Войти»

### Страница `/login`
- Поля: Email, Пароль
- Кнопка: «Войти»
- Ссылка: «Нет аккаунта? Зарегистрироваться»

### Поведение после входа
- **Клиент** → редирект на `/` (каталог)
- **Вендор** → редирект на `/vendor/dashboard`

### Защита роутов
- `/vendor/*` — только для вендоров
- `/client/*` — только для клиентов
- Неавторизованные → редирект на `/login`

### Временно (без бэка)
- Хранить сессию в `localStorage`: `{ id, name, email, role, companyName? }`
- При перезагрузке — восстанавливать сессию

## 6. Страницы и функционал

### 6.1 Публичные страницы (доступны всем)

---

#### `/` — Главная / Каталог вендоров

**Hero-секция:**
- Большой заголовок (Playfair Display): «Найдите лучших вендоров для вашего торжества»
- Подзаголовок: «Площадки, тамады, фотографы, декор и многое другое по всему Казахстану»
- Строка поиска с выбором города

**Категории (горизонтальный скролл):**
- Иконка + подпись для каждой категории
- Клик = фильтр по категории

```
🏛️ Площадки   🎤 Тамада   📷 Фото/Видео
🌸 Декор       🎵 Музыка   🍽️ Кейтеринг   ✨ Другое
```

**Фильтры:**
- Категория (dropdown или pills)
- Город: Almaty, Astana, Shymkent, Aktau, Kostanay
- Цена: от — до (₸)
- Вместимость: минимум гостей
- Рейтинг: от ★ до ★★★★★

**Сетка офферов:**
- 3 колонки на desktop, 2 на tablet, 1 на mobile
- Карточка оффера (см. компонент OfferCard)
- Пагинация или infinite scroll

---

#### `/offers/[id]` — Детальная страница оффера

**Секции:**
1. **Фото-галерея** — сетка или слайдер, до 10 фото
2. **Заголовок блок** — название, вендор, город, категория, рейтинг
3. **Ключевые данные** — цена, вместимость, доступные даты
4. **Описание** — полный текст
5. **Calendario доступности** — визуальный календарь свободных дат
6. **Отзывы** — список отзывов с оценками и датами
7. **Форма заявки** (сайдбар или снизу):
   - Выбрать дату из доступных
   - Ввести имя (или берём из профиля если авторизован)
   - Кнопка «Отправить заявку» → переход в чат

---

#### `/chat/[bookingId]` — Чат между клиентом и вендором

**Шапка чата:**
- Название оффера, имя клиента, дата мероприятия
- Статус заявки: `Ожидает ответа` / `Договорились ✓` / `Отклонено`

**Область сообщений:**
- Пузыри сообщений (свои справа, чужие слева)
- Время отправки под каждым сообщением

**Баннер подтверждения** (когда статус = confirmed):
```
🎉 Вы договорились! Дата мероприятия: 15 июня 2026
```

**Действия вендора** (только если статус = pending):
- Кнопка «Подтвердить сотрудничество» (зелёная)
- Кнопка «Отклонить» (серая/красная)

**Поле ввода:**
- Текстовое поле + кнопка отправки
- Enter = отправить

## 7. Модели данных

### User
```typescript
type User = {
  id: string;
  name: string;
  email: string;
  role: 'client' | 'vendor';
  companyName?: string;   // только для вендоров
  city?: string;
  phone?: string;
  createdAt: string;
};
```

### Offer
```typescript
type Category = 'venue' | 'catering' | 'music' | 'photo_video' | 'decor' | 'tamada' | 'other';
type City = 'Almaty' | 'Astana' | 'Shymkent' | 'Aktau' | 'Kostanay';

type Offer = {
  id: string;
  vendorId: string;
  vendorName: string;
  title: string;
  description: string;
  category: Category;
  city: City;
  price: number;           // стоимость в тенге
  capacity?: number;       // вместимость гостей (для venue, catering)
  photos: string[];        // URL фотографий
  availableDates: string[]; // ISO даты свободных дней
  rating?: number;         // средняя оценка (1–5)
  reviewCount?: number;
  createdAt: string;
};
```

### Booking
```typescript
type BookingStatus = 'pending' | 'confirmed' | 'rejected';

type Booking = {
  id: string;
  offerId: string;
  offerTitle: string;
  vendorId: string;
  clientId: string;
  clientName: string;
  eventDate: string;        // ISO date
  status: BookingStatus;
  messages: Message[];
  createdAt: string;
};
```

### Message
```typescript
type MessageSender = 'client' | 'vendor';

type Message = {
  id: string;
  bookingId: string;
  sender: MessageSender;
  text: string;
  timestamp: string;
};
```

## 10. Флоу пользователей

### Флоу клиента
```
Главная → Фильтрует вендоров → Открывает детальную страницу
→ Выбирает дату в календаре → Нажимает «Отправить заявку»
→ (если не авторизован — логин/регистрация)
→ Переходит в чат → Переписывается с вендором
→ Видит подтверждение сделки
```

### Флоу вендора
```
Регистрация (выбирает роль «Вендор») → Dashboard
→ Создаёт оффер (заполняет форму, загружает фото, отмечает даты)
→ Оффер появляется в каталоге
→ Получает заявку от клиента → Leads inbox
→ Открывает чат → Переписывается → Нажимает «Подтвердить сотрудничество»
→ Сделка подтверждена
```
